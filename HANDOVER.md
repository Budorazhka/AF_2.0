# Aurum Fort — Документация для передачи проекта (Handover Guide)

Этот файл содержит ключевую техническую информацию о проекте для быстрого погружения нового разработчика.

---

## 🛠️ Технологический стек
*   **Основа:** Чистый HTML5 (семантическая разметка) и ванильный JavaScript (ES6+).
*   **Стилизация:** Чистый CSS3 с использованием CSS-переменных (`css/styles.css`).
*   **Инфраструктура:** Docker Compose + Nginx (на базе Alpine Linux) для локального запуска и продакшена.
*   **Анимации и эффекты:** GSAP (GreenSock Animation Platform) + ScrollTrigger.
*   **Плавный скролл:** Lenis.
*   **Бесшовные переходы между страницами:** Barba.js.

---

## 🚀 Локальный запуск
Проект полностью упакован в Docker. Для запуска локального веб-сервера выполните в корне проекта:

```bash
docker compose up -d
```

*   **Порт по умолчанию:** `8090` (настраивается в файле `.env` через переменную `WEB_PORT`).
*   **Адрес в браузере:** [http://localhost:8090](http://localhost:8090)
*   **Монтирование статики:** Папка `./assets` примонтирована в контейнер напрямую (`volumes` в `docker-compose.yml`), поэтому любые новые изображения, шрифты и файлы можно добавлять без пересборки Docker-образа.
*   **Сброс кэша стилей:** Проект использует кэш-бастинг для CSS стилей. При изменении `css/styles.css` не забывайте обновлять версию в подключаемых файлах (например: `href="/css/styles.css?v=45"`).

## ☁️ Деплой на статический хостинг (GitHub Pages / Vercel)

Nginx и `dev_server.py` умеют SSI (`<!--# include virtual="..." -->`) — статические хостинги вроде GitHub Pages и Vercel нет. `scripts/build_pages.py` подставляет партиалы (`partials/top.html`, `partials/bottom.html`) прямо в HTML и складывает результат в `_site/` (не коммитится, генерируется каждый раз заново). Каждая страница пишется и как `name.html`, и как `name/index.html` — чтобы `/location` работало независимо от того, умеет хостинг раскрывать URL без расширения.

*   **GitHub Pages:** `.github/workflows/pages.yml` гоняет сборку в Actions при пуше в `main`. В настройках репозитория Settings → Pages → Source нужно выбрать «GitHub Actions» (не «Deploy from a branch»).
*   **Vercel:** `vercel.json` уже задаёт `buildCommand`/`outputDirectory` — при импорте репозитория достаточно нажать Deploy, никаких доп. настроек не требуется.
*   **Локальная проверка сборки:** `python scripts/build_pages.py`, затем `cd _site && python -m http.server 8099`.

## 🏗️ Архитектура JS и жизненный цикл (Barba.js + GSAP)
Вся логика интерактивности находится в файле [js/main.js](file:///Users/egor/Desktop/aurum/js/main.js). Из-за использования **Barba.js** переходы происходят без полной перезагрузки страницы (AJAX-запросы).

### Важное для разработки:
1.  **Инициализация плагинов:** Все библиотеки (ScrollTrigger, Lenis, слайдеры, анимации появления) должны корректно переинициализироваться при смене страниц. В коде это реализовано через `initPage()` и хуки Barba.js.
2.  **ScrollTrigger / Parallax:** При переходе на новую страницу старые инстансы ScrollTrigger уничтожаются (`ScrollTrigger.getAll().forEach(t => t.kill())`), после чего создаются новые.
3.  **Динамический DOM:** Элементы, создаваемые на лету через JS (например, переходы к следующим главам `#chapterNext`), должны быть полностью встроены в DOM *до того*, как будет запущен поиск элементов под параллакс. Иначе параллакс зацепится за родительский контейнер, что приведет к смещению секций и появлению белых швов у футера.

## 🌐 Двуязычие RU / KA (грузинский)

Сайт двуязычный: русский — базовый (лежит прямо в разметке), грузинский подставляется на лету. Переключатель **РУ / ქარ** — в шапке и в оверлей-меню.

*   **Движок:** [js/i18n.js](js/i18n.js). Читает атрибуты `data-i18n` (textContent), `data-i18n-html` (innerHTML — для строк с `<br>`/`&nbsp;`) и `data-i18n-attr="attr:ключ;attr2:ключ2"` (переводимые атрибуты — `placeholder`, `aria-label`, `alt`, `title`, `content` у meta, `data-chapter`). Ключ перевода — это оригинальная русская строка. Выбор языка хранится в `localStorage('af-lang')`, ставится атрибутом `lang` на `<html>`, переприменяется после каждого перехода Barba.js.
*   **Словарь:** [js/i18n-ka.js](js/i18n-ka.js) — объект `window.AF_I18N_KA = { "русский ключ": "грузинский перевод" }`. Ключи и в разметке, и в словаре прогоняются через одну нормализацию `keyOf()` (снимает разметку, декодирует `&nbsp;`/`&mdash;`, приводит апострофы), поэтому вариации оформления не ломают совпадение. **Чтобы добавить/поправить перевод — редактируется только этот файл** (или добавляется `data-i18n` на новый узел в HTML). Если перевода нет — грациозно показывается русский.
*   **Шрифты:** Tenor Sans и Montserrat не содержат грузинских глифов. При `html[lang="ka"]` переменные `--serif`/`--sans`/`--body` в [css/styles.css](css/styles.css) переключаются на **Noto Serif Georgian** (дисплей/заголовки) и **Noto Sans Georgian** (текст). Оба семейства подключены в `<head>` каждой страницы рядом с латинскими.
*   **JS-строки** (названия глав в блоке «Следующая глава») локализуются в [js/main.js](js/main.js) через `AF_I18N.t()`; при смене языка блок обновляется через `window.AF_refreshChapterNext()` без полной пересборки.
*   **Кэш-бастеры:** CSS `?v=55`, main.js `?v=18`, i18n-скрипты `?v=1`. При правках не забывать поднимать версию.
*   **Проверка:** ключи разметки и словаря сверяются скриптом (все 230 совпадают); связка протестирована в Playwright (переключение, шрифты, сохранение выбора, отсутствие JS-ошибок).
