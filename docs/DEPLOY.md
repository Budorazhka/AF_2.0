# DEPLOY — деплой Aurum Fort на VPS

Статический сайт, один контейнер nginx, без бэкенда и базы — деплой заметно проще,
чем у обычного full-stack приложения.

**Пока без домена**: на боевом домене сейчас висит тестовая версия сайта — её
не трогаем. Сайт поднимается на VPS по IP на отдельном порту, домен подключается
отдельным шагом, когда решите переключаться (см. раздел 6).

## 1. На VPS — установить Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# перелогиниться, чтобы группа применилась
```

## 2. Клонировать проект

```bash
git clone https://github.com/Budorazhka/AF_2.0.git aurum
cd aurum
```

## 3. Создать `.env`

Файл в git не попадает (`.gitignore`), после клонирования его нет — а
`docker-compose.yml` ждёт его как `env_file`. Секретов внутри нет, только порт:

```bash
echo "WEB_PORT=8090" > .env
```

Если порт `8090` на VPS уже занят — указать свободный.

## 4. Запустить

```bash
docker compose up -d --build
```

Проверить:
```bash
docker compose ps                      # web — healthy/running
curl -I http://localhost:8090/
```

Открыть в браузере: `http://<IP-вашего-VPS>:8090` — домен трогать не нужно, сайт
живёт отдельно от того, что сейчас на нём висит.

Если порт `8090` на файрволе VPS закрыт снаружи — открыть его
(`ufw allow 8090/tcp` или аналог в вашем файрволе), либо смотреть сайт через
`ssh -L 8090:localhost:8090 user@vps` и `http://localhost:8090` у себя.

## 5. Обновление до новой версии

```bash
cd aurum
git pull
docker compose up -d --build
```

Картинки лежат в `assets/img` как volume (см. `docker-compose.yml`) — их можно
докидывать без пересборки образа, `docker compose restart web` достаточно.

## 6. Когда будете готовы переключить домен

Не делаем сейчас, просто чтобы был план:

1. Освободить домен от тестовой версии (снять то, что сейчас там крутится).
2. Поставить перед контейнером reverse-proxy с HTTPS — проще всего Caddy
   (сам тянет Let's Encrypt):
   ```
   ваш-домен.example.com {
       reverse_proxy localhost:8090
   }
   ```
3. `sudo systemctl reload caddy`.

## 7. Логи

```bash
docker compose logs -f web
docker compose logs --tail 200 web
```

## 8. Откат

```bash
git log --oneline           # найти прошлый рабочий коммит
git checkout <старый-коммит>
docker compose up -d --build
```

База/миграции не задействованы — откат это просто откат кода и пересборка образа.

## Чек-лист перед переключением домена

- [ ] Сайт отвечает по IP на выбранном порту, `docker compose ps` — healthy
- [ ] Тексты-плейсхолдеры и контакты в `index.html`/`.env` заменены на реальные
      (см. README.md → «Что доделать под клиента»)
- [ ] Отправка формы обратной связи подключена (сейчас `TODO` в `js/main.js`)
- [ ] Заглушки `data-placeholder` (видео с дрона, фото конструктива, цитата
      архитектора) заменены на реальные материалы — искать по этому атрибуту
